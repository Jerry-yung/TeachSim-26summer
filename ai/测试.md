# TeachSim AI 联调测试手册

以下命令均在 `**ai**` 目录下执行（保证 [教案](教案/) 目录下相对路径有效）。默认服务地址为 `**http://localhost:8001**`（与 `python main.py` 默认一致）。字段口径见 [docs/ai-backend-contract-v2.md](docs/ai-backend-contract-v2.md)。

**命令格式说明**

- 使用 **反斜杠 `\` 换行**，或使用 `**<<'EOF' ... EOF'`** 传递 JSON，在 **bash / zsh** 下可整段复制到终端执行。
- 大段 JSON 建议用 `**--data-binary @tests/samples/xxx.json`**，避免粘贴损坏；示例见 [tests/samples/](tests/samples/) 目录（如 [segment_eval_body.json](tests/samples/segment_eval_body.json)、[report_generate_body.json](tests/samples/report_generate_body.json)）。

## 0. 启动 AI 服务（后端）

1. 进入本目录并安装依赖（首次）：

```bash
# 先进入 仓库根目录，再进入 ai（若你已在 ai 目录可跳过 cd）
cd ai
pip install -r requirements.txt
```

1. 在项目根目录配置 `.env` 中的大模型 API Key（与 [setting/llm.py](setting/llm.py) 所用供应商一致）。
2. 启动服务（默认监听 `0.0.0.0:8001`）：

```bash
python main.py
```

1. 另开终端，在 `**ai**` 目录下验证服务已起来：

```bash
curl -s http://localhost:8001/health
```

**示例输出：**

```json
{
  "status": "ok",
  "service_url": "http://localhost:8001",
  "timestamp_ms": 1713200000000
}
```

看到包含 `"status":"ok"` 即可继续下面测试。

说明：若需改端口，可设置环境变量 `AI_PORT` 后重启，并把下文所有 `localhost:8001` 改成你的端口。

## 4. SupervisorAgent 测试（是否调用 StudentAgent）

**接口：** `POST /ai/v2/inclass/supervisor/decide`

**请求体格式（新格式，与 `main.py` 中 `SupervisorV2Request` 一致）：**

- `teacher_text`（string, required）：教师**本轮**发言。
- `class_elapsed_sec`（int, optional，建议传）：课内计时器已过秒数；服务端会格式化为 `MM:SS` 作为教师本轮时间标签传入 Agent（与 `docs/ai-backend-contract-v2.md` 中的 ISO8601 示例不同，以代码为准）。
- `subject`（string, optional）：学科（如"初中数学"）。
- `chat_history`（array, optional）：对话历史，包含教师和学生发言。每项含 `role`（"teacher"|"student"）和 `content`（发言内容）。
- `current_ppt`（array, optional）：**仅传当前页一条**，结构与 **2.2 PPT 结构化** 输出中 `slides[]` **单条**一致。
- `called_student_status_digest`（object | string | null, optional）：被点名学生状态摘要；无摘要时传 JSON `null`。与「提问类」组合时：`null` → `questioning`，非 `null` → `relay_answer`。

**不要**传入 `dialog_state`：由 `SupervisorAgent` 内部用 LLM 推断（失败时规则回退），再据此决定是否在本次响应中生成 `student_event`。

说明：`tests/samples/supervisor_v2_decide_*.json` 中若仍写有 `current_timestamp`，当前实现**不会读取**该字段（Pydantic 忽略未知字段）；联调若要时间标签生效，请在 JSON 中改为 `class_elapsed_sec`（整数秒）。

服务端先用 **LLM** 推断 `dialog_state`（失败时用规则）；再按状态映射 `should_trigger_student` 等。响应里会带上 `**dialog_state`**，取值含：`normal` | `questioning` | `relay_answer` | `ambiguous` | `misstatement` | `discipline_whisper` | `discipline_sleep`。

规则摘要（与 `inclass_supervisor_agent.py` 中 `decide()` 一致）：

- `normal`：`student_event` 为 `null`，不调用 `StudentAgent`。
- `questioning`：`should_trigger_student=true`，`target_student_type="all"`，但 `**student_event` 固定为 `null`**——**本次不调用** `StudentAgent` 预生成候选；前端点名后改调 `**POST /ai/v2/inclass/student/reply`** 按需生成单条回复。Supervisor 在同一次状态推断中与 `dialog_state` 一并输出 `**question_difficulty`**（整数 1-5，5 最难；非 questioning 时为 null）。
- `relay_answer`：内部调用 `StudentAgent`，`student_event` 为 **长度为 1 的数组**（`called_student_status_digest.student_type` 对应角色，`is_proactive_speaking=true`）。
- `ambiguous` / `misstatement` / `discipline_whisper` / `discipline_sleep`：内部调用 `StudentAgent`，`student_event` 为 **单个对象**；其中仅 `**ambiguous` / `misstatement`** 的 `is_triggered` 与 `is_proactive_speaking` 为 `true`，**纪律类两项均为 `false`**（见 `inclass_supervisor_agent.py` 中 `decide()`）。

### 4.2 推断为 questioning（不预生成 student_event，点名后走 reply）

请求体：[tests/samples/supervisor_v2_decide_questioning_body.json](tests/samples/supervisor_v2_decide_questioning_body.json)（可将其中 `current_timestamp` 改为 `class_elapsed_sec`，例如 `388` 表示 06:28）

说明：该场景下 `**/supervisor/decide` 不会调用 `StudentAgent**`，`student_event` 恒为 `null`。前端在点名某位学生后，使用 `**POST /ai/v2/inclass/student/reply**` 传入 `student_type` 与 `background` 生成**一条**回复；`is_proactive_speaking` 可按产品需要传 `true`/`false`。

```bash
curl -s -X POST "http://localhost:8001/ai/v2/inclass/supervisor/decide" \
  -H "Content-Type: application/json" \
  -H "X-Trace-Id: trace-supervisor-questioning-001" \
  --data-binary @tests/samples/supervisor_v2_decide_questioning_body.json
```

**示例输出（与代码一致）：**

```json
{
  "dialog_state": "questioning",
  "should_trigger_student": true,
  "trigger_reason": "teacher_question",
  "target_student_type": "all",
  "student_event": null,
  "question_difficulty": 3
}
```

**补充：点名后生成单条学生回复（`StudentAgent`）**

```bash
curl -s -X POST "http://localhost:8001/ai/v2/inclass/student/reply" \
  -H "Content-Type: application/json" \
  -H "X-Trace-Id: trace-supervisor-questioning-reply-001" \
  -d '{
    "student_type": "xueyou",
    "trigger_reason": "teacher_question",
    "is_proactive_speaking": true,
    "background": {
      "subject": "初中数学",
      "slide_no": 2,
      "slides": [{
        "slide_no": 2,
        "title": "勾股定理与前提",
        "text_blocks": ["勾股定理仅适用于直角三角形", "a²+b²=c²"],
        "visual_elements": ["直角三角形示意图"],
        "summary": "定义与适用条件"
      }],
      "teacher_utterances_on_slide": [
        {"ts": "06:28", "text": "谁能解释一下为什么勾股定理不能用于任意三角形？"}
      ]
    }
  }'
```

`reply_text` 等字段由 **LLM** 生成，以实际返回为准。

## 6. 验收清单

- 上述命令在 **ai** 目录、服务已启动时可直接执行。
- 教案解析无 `preset_questions` 字段。
- Supervisor 请求体新格式：`teacher_text` + `class_elapsed_sec`（建议）+ `subject` + `chat_history` + `current_ppt` + `called_student_status_digest`（无摘要时传 `null`）。
- Supervisor：`questioning` 返回 `student_event: null`，**不**在 `decide` 内调用 `StudentAgent`，点名后走 `**/ai/v2/inclass/student/reply`**；`relay_answer` 返回长度为 1 的 `student_event` 数组（与 `called_student_status_digest.student_type` 一致，`is_triggered=false`，`is_proactive_speaking=true`）；`ambiguous`/`misstatement` 返回单个 `student_event` 对象（`is_triggered=true` 且 `is_proactive_speaking=true`）；`normal` 为 `student_event: null`。
- `relay_answer` 场景需在 `chat_history` 中包含学生发言记录，且 `called_student_status_digest` 为非 `null`。
- 报错时见控制台或返回体中的 `detail`（FastAPI 包装）。
