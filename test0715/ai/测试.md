# TeachSim AI 联调测试手册

以下命令均在 `**ai**` 目录下执行（保证 [教案](教案/) 目录下相对路径有效）。默认服务地址为 `**http://localhost:8001**`（与 `python main.py` 默认一致）。字段口径见 [docs/ai-backend-contract-v2.md](docs/ai-backend-contract-v2.md)。

**命令格式说明**

- 使用 **反斜杠 `\` 换行**，或使用 `**<<'EOF' ... EOF'**` 传递 JSON，在 **bash / zsh** 下可整段复制到终端执行。
- 大段 JSON 建议用 `**--data-binary @tests/samples/xxx.json`**，避免粘贴损坏；示例见 [tests/samples/](tests/samples/) 目录（如 [segment_eval_body.json](tests/samples/segment_eval_body.json)、[report_generate_body.json](tests/samples/report_generate_body.json)）。

## 0. 启动 AI 服务（后端）

1. 进入本目录并安装依赖（首次）：

```bash
# 先进入 TeachSim0416 仓库根目录，再进入 ai（若你已在 ai 目录可跳过 cd）
cd ai
pip install -r requirements.txt
```

1. 在项目根目录配置 `.env` 中的大模型 API Key（与 [setting/llm.py](setting/llm.py) 所用供应商一致）。
2. 启动服务（默认监听 `0.0.0.0:8001`）：

```bash
python main.py
```

1. 另开终端，在 `**ai**` 目录下验证服务已起来：

```bash
curl -s http://localhost:8001/health
```

**示例输出：**

```json
{
  "status": "ok",
  "service_url": "http://localhost:8001",
  "timestamp_ms": 1713200000000
}
```

看到包含 `"status":"ok"` 即可继续下面测试。

说明：若需改端口，可设置环境变量 `AI_PORT` 后重启，并把下文所有 `localhost:8001` 改成你的端口。

## 4. SupervisorAgent 测试（是否调用 StudentAgent）

**接口：** `POST /ai/v2/inclass/supervisor/decide`

**请求体格式（新格式）：**
- `teacher_text`（string, required）：教师**本轮**发言。
- `current_timestamp`（string, optional 但**强烈建议必传**）：ISO8601（含时区），表示当前时刻。
- `subject`（string, optional）：学科（如"初中数学"）。
- `chat_history`（array, optional）：对话历史，包含教师和学生发言。每项含 `role`（"teacher"|"student"）和 `content`（发言内容）。
- `current_ppt`（array, optional）：**仅传当前页一条**，结构与 **2.2 PPT 结构化** 输出中 `slides[]` **单条**一致。
- `called_student_status_digest`（object | string | null, optional）：被点名学生状态摘要；无摘要时传 JSON `null`。与「提问类」组合时：`null` → `questioning`，非 `null` → `relay_answer`。

**不要**传入 `dialog_state`：由 `SupervisorAgent` 内部用 LLM 推断（失败时规则回退），再据此决定是否调用 `StudentAgent`。

服务端先用 **LLM** 推断 `dialog_state`（失败时用规则）；再按状态映射是否触发学生。响应里会带上 `**dialog_state`**，取值含：`normal` | `questioning` | `relay_answer` | `ambiguous` | `misstatement` | `discipline_whisper` | `discipline_sleep`。

规则摘要：`normal` 不调用 `StudentAgent`；其余状态会调用并填充 `student_event`。其中 `questioning` 返回 6 条事件（学优主动/非主动、杠精主动/非主动、学困主动/非主动）；`relay_answer` 仅返回 1 条事件（`called_student_status_digest.student_type` 对应角色，`is_proactive_speaking=true`）；`ambiguous/misstatement` 返回单个学生事件（主动发言）。

## 6. 验收清单

- 上述命令在 **ai** 目录、服务已启动时可直接执行。
- 教案解析无 `preset_questions` 字段。
- Supervisor 请求体新格式：`teacher_text` + `current_timestamp` + `subject` + `chat_history` + `current_ppt` + `called_student_status_digest`（无摘要时传 `null`）。
- Supervisor：`questioning` 返回 6 条 `student_event`（学优主动/非主动 + 杠精主动/非主动 + 学困主动/非主动，`is_triggered=false`）；`relay_answer` 返回 1 条 `student_event`（与 `called_student_status_digest.student_type` 一致，`is_triggered=false`，`is_proactive_speaking=true`）；`ambiguous/misstatement` 返回 1 条 `student_event`（`is_triggered=true` 且 `is_proactive_speaking=true`）；`normal` 无 `student_event`。
- `relay_answer` 场景需在 `chat_history` 中包含学生发言记录，且 `called_student_status_digest` 为非 `null`。
- 报错时见控制台或返回体中的 `detail`（FastAPI 包装）。
